CONSTRUTOR DE CÓPIA
- O construtor de cópia é um construtor especial chamado quando um objeto pe criado a partir de outro objeto da mesma classe.
- Ele é invocato automaticamente em três situações principais:
    - Passagem por valor
        - Quando um objeto é passado como parâmetro por valor para uma função, uma cópia é criada.
        - O construtor da cópia é responsável por inicializá-la
    - Retorno por valor
        - Quando uma função retorna um objeto por valor, uma cópia do objeto retornado é criado para o chamador
    - Inicialização de novo objeto
        - Quando um novo objeto é inicializado com outro (ex.: PilhaInt p2 = p1;)

- Se nenhum construtor de cópia for declarado, será criado um default que aplicará o construtor de cópia a cada dado do membro, restando três opções:
    - Usar o default
        - Avalie cuidadosamente se o comportamento de cópia membro a membro é adequado para sua classe
    - Escrever um próprio
        - Implemente um construtor de cópia personalizado quando a cópia superficial não for suficiente (ex.: classe com ponteiros)
    - Desativá-lo
        - Construtor de cópia como private para impedir que o objeto seja copiado (útil para recursos únicos)

EXEMPLO DE CONSTRUTOR DE CÓPIA
class PilhaInt
{ 
  public:
    PilhaInt();
    PilhaInt( const PilhaInt& p );
    void empilha( int valor );
    int desempilha();
  private:
    int tab[MAX_PILHA];
    int atual;
};

- Parâmetro por referência
    - O construtor de cópia semre recebe o parâmetro por referência (&).
- Qualificador const
    - O qualidicador const garante que o objeto fonte não será modificado durante a cópia. É boa prática e permite receber objetos temporários (rvalues)
- Implementação
    - A implementação deve copiar todos os dados relevantes: o array "tab" e o índice "atual", garantindo que a cópia seja independente do original

OPERADOR DE ATRIBUIÇÃO

- O operador = é usado para copiar objetos do mesmo tipo quando ambos já existem. Se difere do construtor de cópia porque age sobre um objeto já inicializado
- Características Importantes
    - O construtor de cópia cria um novo objeto; o operador = age sobre um objeto já existente
    - Deve retornar uma referência constante ao objeto passado como parâmetro (const PilhaInt&)
    - Retorno por valor não é uma boa ideia - cria uma cópia desnecessária
    - Se não for definido, haverá uma cópia membro a membro dos dados (default)

- Retornar uma referência não constante tornaria (A = B) = C válido sintaticamente, mas sem sentido semântico
- Retornar uma referência constante impede o uso incorreto e ainda permite o encadeamento natural A = B = C

OBS: Sempre verifique se "this != &p" antes de copiar para evitar auto-atribuição problemática: "if (this == &p) return *this;"

Implementação do Operador =
    class PilhaInt 
    { 
    public:
    ...
        const PilhaInt& operator= ( const PilhaInt& p ){
            if( this != &p ) {
                atual = p.atual;
                for( int i = 0; i < p.atual; i++ ) {
                    tab[i] = p.tab[i];
                }
            }
            return p;
        }
    }

- Assinatura: "operator =" é a sintaxe C++ para sobrecarregar o operador de atribuição.
- Cópia dos dados: Copia o íntice atual e, em seguida, copia elemento a elemento o conteúdo do array tab - apenas os elementos válidos
- O ponteiro this: "this" é um ponteiro para o próprio objeto (o objeto do lado esquerdo da atribuição). "return p" retorna uma referência ao prórpio objeto passado como parâmetro, permitindo encadeamento de atribuições
